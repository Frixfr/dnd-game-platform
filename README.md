# D&D Assistant – игровое приложение (мастер + игроки)

Система для управления настольными ролевыми играми в стиле D&D.  
Позволяет мастеру создавать игроков, NPC, расы, способности, эффекты, предметы и управлять боем.  
Игроки могут использовать активные способности, следить за персонажем и видеть карту (в разработке).

## Технологии

- **Фронтенд**: React (Vite), TypeScript, Tailwind CSS, Zustand, Socket.IO client
- **Бэкенд**: Node.js (Express), TypeScript, SQLite (knex), Socket.IO
- **Хранение файлов**: локальная файловая система (аватарки)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd <папка-проекта>
```

### 2. Настройка переменных окружения

Скопируйте пример переменных окружения:

```bash
cp .env.example .env
```

При необходимости отредактируйте `.env` (порт, хост, адрес клиента).  
Для разработки значения по умолчанию подходят.

### 3. Установка зависимостей

Проект разделён на две независимые части: `client/` (фронт) и `server/` (бэк).  
Установите зависимости для каждой:

```bash
cd client
npm install

cd ../server
npm install
```

### 4. База данных (SQLite)

Бэкенд использует SQLite – **не требует установки отдельного сервера БД**.  
При первом запуске сервера автоматически создадутся:

- файл базы данных `server/data/game.db`
- все необходимые таблицы (миграции через `server/src/db/init.ts`)

Если потребуется сбросить БД – просто удалите файл `server/data/game.db` и перезапустите сервер.

### 5. Запуск в режиме разработки (dev)

**Важно:** запускать сервер и клиент нужно одновременно, но в разных терминалах.

**Терминал 1 – бэкенд:**

```bash
cd server
npm run dev
```

Сервер запустится на `http://localhost:5000` (или порт из `.env`).

**Терминал 2 – фронтенд:**

```bash
cd client
npm run dev
```

Клиент запустится на `http://localhost:3000`.  
Vite автоматически проксирует API-запросы на бэкенд (настройка в `vite.config.ts`).

> **Примечание:** если вы хотите запускать обе части одной командой (например, через `concurrently`), вы можете добавить корневой `package.json` с соответствующим скриптом. В текущей версии такого скрипта нет.

### 6. Production-сборка

**Сборка клиента:**

```bash
cd client
npm run build
# собранные файлы появятся в client/dist
```

**Сборка сервера:**

```bash
cd server
npm run build
# транспилированный код в server/dist
```

**Запуск сервера в production:**

```bash
cd server
npm start
```

Для production также потребуется настроить переменную `CLIENT_ORIGIN` в `.env` на реальный адрес, с которого будут приходить запросы (например, `https://your-domain.com`).  
Статику клиента можно отдавать через Nginx или любой другой веб-сервер.

## Использование

### Вход в систему

- **Мастер:** на странице входа выберите «Мастер», введите пароль `dm123` (хардкод, временная аутентификация).
- **Игрок:** выберите «Игрок», введите имя персонажа (любое) и выберите одну из доступных карточек игроков (у которых `is_card_shown = true` и не установлен пароль, либо введите пароль, если он задан мастером).

### Основные возможности мастера

- CRUD игроков, NPC, рас, способностей, эффектов, предметов.
- Управление активными эффектами (добавление/удаление).
- Экипировка предметов для игроков и NPC.
- Боевая система: инициатива, ходы, раунды, уменьшение кулдаунов и длительности эффектов в конце раунда.
- Просмотр логов действий (плавающая кнопка).
- Завершение дня (уменьшение `remaining_days` эффектов) – через отдельный API (пока без UI).

### Возможности игрока

- Просмотр своего листа персонажа (только чтение).
- Использование активных способностей (кнопка «Использовать»).
- Просмотр инвентаря, эффектов, карты (заглушка).

## Структура проекта

Полная структура файлов описана в документации проекта (файлы `Структура файлов бэк.txt`, `Структура файлов фронт.txt`, `Ключевые правила.txt`).

Ключевые директории:

- `client/` – React-приложение (Vite)
- `server/` – Express + Socket.IO + SQLite
- `server/data/` – файл базы данных (создаётся автоматически)
- `server/uploads/avatars/` – загруженные аватарки

## Примечания по разработке

- **Пересчёт статов** всегда происходит на бэкенде (функции `calculateFinalStats`, `getFullPlayerData`). Фронт только отображает `final_stats`.
- **Расы** дают эффекты через таблицу `race_effects`.
- **Пассивные способности** при добавлении игроку автоматически создают запись в `player_active_effects`.
- **Экипированные предметы** (`is_equipped=1`) влияют на статы через `passive_effect_id`.
- **Активные эффекты** имеют `remaining_turns` (уменьшается в конце раунда боя) и `remaining_days` (уменьшается только по команде мастера «завершить день»).
- **Удаление сущностей** с проверкой зависимостей (409 Conflict при наличии связей).
- **Socket.IO** события: `entity:action` (например, `player:created`, `combat:updated`). При `player:updated` всегда передаётся полный объект с `final_stats`.
